Section 02

기획의도: 시대적 배경 Context & Intention

2022년 ChatGPT를 기점으로 LLM이 폭발적으로 발전하면서, 개발 도구가 산업재에서 소비재로 전환되고 있습니다.

Andrej Karpathy가 2025년 명명한 '바이브코딩(Vibe Coding)'은 이 전환의 상징 — 비전문가도 자연어 지시만으로 실제 작동하는 소프트웨어를 만들 수 있는 시대.

> "산업재였던 개발이, 소비재가 된 시대
   누구나 말로 만든다"
PROBLEM
대부분의 개발 채널은 여전히 기술 중심적. '개발의 민주화'를 대중의 언어로 전달하는 콘텐츠는 부재
SOLUTION
기술의 민주화를 유머로 증명하는 채널. 바이브코딩 × 예능이라는 니치 = 블루오션
POSITION
개발 채널 (진지) ← 쓸데없는앱연구소 → 예능 채널 (코딩 無)

문화원형 & 컨셉 방법론 Archetype & BAD Method

= 쓸데없는앱연구소

Jung의 12가지 archetype 중 Jester(유머와 현재를 즐기려는 욕구)와 Creator(새로운 것을 만들려는 욕구)의 교차점에 위치합니다.

Section 03

포맷 상세 Format Details

양성희(2013)의 포맷 3요소 프레임워크 기반

FORMAT 01
변하지 않는 그릇
인트로: 터미널 타이핑 $ 오늘의 하찮은 문제:

시그니처 대사: "대딸깍의 시대, 제가 해결해보겠습니다"

엔딩: 완성된 앱 QR코드 + 클로징

BGM: lo-fi 코딩 비트
자막: 코드 에디터 폰트 + 초록색 하이라이트
FORMAT 02
구조화된 서사 형식
매 에피소드 5단계 구조:

1. 문제 발견 (1~2분)
2. 기획 회의 (1분)
3. 바이브코딩 (3~5분)
4. 결과물 시연 (2분)
5. 평가 & 클로징 (1분)

시청자가 문제~해결 여정에 동참
FORMAT 03
소재 소화력
"일상의 하찮은 문제" 프레임은
소재가 무한합니다.

음식 • 연애 • 학교 • 날씨
건강 • 쇼핑 • 교통 • 직장
+ 무한...

반복 가능성과 확장 가능성을 모두 충족 (양성희, 2013)

긱블 — "엔지니어링 예능"
유사: 기술을 예능으로, "만들기"가 핵심
차이: 긱블=하드웨어/팀, 본 채널=소프트웨어/1인

REFERENCE 03
피식대학 — "블루오션 전략"

EP.01 구성안 Episode 01 Rundown

"친구들이랑 약속 잡을 때 정확히 중간지점 찾아주는 앱 만들어봤습니다" — 총 5분 30초

인트로 0:00~0:15 터미널 타이핑 + 문제 텍스트 출력 터미널 화면 + PIP 얼굴캠 타이핑 효과음 + lo-fi

기획 1:05~1:35 "대딸깍의 시대" + 앱 기능 스케치 화이트보드 + 얼굴캠 아이디어 효과음
